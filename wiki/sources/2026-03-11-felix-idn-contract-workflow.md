# IDN Contract Workflow & Pricing System — Felix Rode

Granola meeting: `6ae538bd-daf4-415c-a4b7-b7a097c4731f`

## Summary

Deep dive with Felix Rode (Sr. Director Strategic Pricing & Contracting, Beckman). Current CPQ system is 12–15 years old, built in isolation, held together by "shoestrings and duct tape." McKinsey's SKU-centric approach may not fit Beckman's solution-based selling.

## System Architecture Pain Points

- Oracle CPQ heavily customized — can't use built-in price AI
- Document management (EDMS) for contracts + Oracle for pricing — not integrated
- ~10 different contract records in Oracle against the same paper
- Standard work scattered across hard drives, shared drives, SharePoint

## Workflow

1. 10–15 min to input customer data → proposal/contract generated
2. System picks correct template (standard Beckman or pre-negotiated boilerplate)
3. DocuSign for execution
4. Negotiation path: Word doc for editable version → custom document
5. IDN: individual iQuotes → price aggregator (Excel) → finance business analysis → margin analysis → contract design

## Billing Complexity

- Four purchase modalities: purchase, lease, reagent rentals, billable programs
- Different billing across product lines — "we're selling solutions, not screws"

## Legacy Contract Issues

- 97-page contracts with conflicting terms
- 10+ year old agreements with multiple amendments
- Heavy manual copy-paste between systems

## Felix's Skepticism

- "If you think you can do this, that's fine. But I don't think this is going to be as straightforward as they think it is"
- Dirty source data with too many permutations
