# Enterprise AI Team Workshop Debrief & Next Steps

Post-training debrief session for the 2-day LBS on-site workshop in San Diego. Morning session with brainstorming + feedback, followed by Anish/Keith demos, then Madina's EAI portion. Team offsite at 1:30 PM.

## Training Feedback (Detailed)

### What Worked
- Getting the team together in person was valuable
- Cross-site visibility (Melbourne, Shanghai, San Diego) led to organic knowledge sharing
- Anish/Keith demos inspired the team more than the formal training

### What Didn't Work
- **Too much lecture, not enough hands-on** — first day was mostly slides/talking
- **Trainer used pre-configured environment** — his prompts worked in his setup but not in attendees' environments
- **"Secret sauce" problem** — trainer mentioned agent swarming but wouldn't teach it ("that's my secret sauce")
- **Generic content** — trainer's examples weren't relevant to LBS's specific tech stack
- **Not progressive** — should have been "build step 1, now use it in step 2" etc.
- **Identical outputs** — when everyone ran the same prompt, results varied wildly; trainer's worked perfectly (likely had history/context cached)

### Constructive Suggestions from Team
- "Take a piece of it, talk about it, then let's build it and keep building progressively"
- Want real project kaizen-style training, not lecture format
- Different skill levels need different tracks (beginners vs power users)
- Virtual classrooms with Microsoft could handle intro-level content

## AI Adoption Infrastructure

### Token/Licensing Model
- AI tools run on token-based cost model
- Subscriptions come with set limits; can request more
- Enterprise AI team covers costs
- Some team members didn't understand the token/cost concept — needs better communication

### Documentation & Process Pain Points
- Documentation repeatedly cited as biggest opportunity
- Teams adding AI references to existing docs
- Quality/regulatory teams interested in AI for compliance docs
- One team member described process as "broken" — can't find information they used to find easily

## Action Items from Session

- [ ] Explore Claude Code pilot as power-user discovery mechanism
- [ ] Team to reconvene in ~1 month for agent design patterns discussion
- [ ] Investigate documentation agent opportunities for LBS
