# GitHub Copilot Adoption

Enterprise-wide deployment of GitHub Copilot to 3,000+ software developers at [[organizations/danaher|Danaher]]. Part of the "Danaher Brain" AI strategy presented to the board.

## Targets

- **95% monthly active users** (Tom Neufelder's floor)
- **80% weekly active users** (strong bar)
- **15% productivity gain in 2026** = $40M+ reinvested productivity
- **2030 ambition**: 70-100% productivity gain

## Current State (as of Apr 2026)

- Less than half of 3,000 license holders have ever logged in
- Beckman at ~30% despite claiming "AI leaders" for a year
- Cursor users (Keith, Anish) delivering ~2x productivity but not yet captured in dashboard
- Productivity metric: lines of code saved at $50/hr assumed rate
- LBS at 42% adoption; IDT currently leading
- Dashboard sent to DLT — low numbers sparked executive attention

## Strategy

- Top-down mandate + bottom-up enablement
- "Gemba walk" approach: identify users and non-users, understand why
- Remove roadblocks (budget, IT, legal), enable training, surface best use cases
- Escalation path: 20%+ refusal after reasonable steps -> escalate to DLT
- Julie emailed OpCo presidents: "if someone isn't using the tools, assume they're less productive"
- License accountability: quarterly productivity requirements, monthly checkpoints, removal after one quarter of inactivity

## LBS On-Site Training Week (Apr 7-10, 2026)

Major training event in San Diego. Key takeaways:
- **Training format doesn't work** — too much lecture, not enough hands-on; trainer's "secret sauce" withholding frustrated attendees
- **Developers want tool choice** — "meet us where we are"; many don't use GitHub, find Copilot limiting
- **Secret AI usage** — Madina 200% certain people are using non-sanctioned AI tools (Claude Code, Cursor, etc.)
- **Bryan Conn** — non-developer power user; built Salesforce MCP integration, image quality tool, ImageScope revival plan; strongest non-developer AI adoption story at Danaher
- **Anish FDA demo** — built full 40,000-slide FDA submission automation with Cursor; $350 in API credits vs months of manual work
- **Claude Code pilot** — Martin suggested as power-user discovery mechanism (self-selection like Cursor pilot)
- **GitHub migration** — LBS code split across Perforce, BitBucket, GitLab; migration blocked by QA validation requirements
- **Copilot architecture** — orchestrator, not LLM itself; MCP enables BitBucket/GitLab integration
- **Licensing clarity** — Business: 300 premium req/mo; Enterprise: 1,000/mo; overage ~$0.04/req
- **AI spending tracked** — Cursor leaderboard shows usage; Blaze (corporate) leading at 71K; LBS team 3rd-4th at 40K+
- **Infrastructure** — $600K allocated for 2026; need 25 GPUs; $150K short; exploring Nvidia SaaS vs hardware ownership
- **Rovo gap** — turned off after enterprise consolidation; team wants it back
- **Direct API access** — real need beyond IDE tools; some devs paying out of pocket; need to structure IDE vs API access
