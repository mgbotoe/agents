### Architecture decisions made

- **Cloud-cron over local Task Scheduler.** Memory pipeline now runs on GitHub Actions; computer-on dependency removed. Email-on-failure fixes the silent-failure problem that bit on May 4–5.
- **Per-repo isolation for SAMESF.** Different domain (marketing) + different lifecycle vs personal infra agents. Independent failure mode. Two repos, copy-paste promote scripts (~60 lines each), no shared package — premature abstraction otherwise.
- **Discussion workflow over inbox/respond pattern.** Original Phase 1 design (turn-based message inbox) was overengineered for "agents talk while I sleep" use case. Single workflow run drives the whole conversation, one transcript file = one commit. Inbox/respond can be added later if needed.
- **Daily-logs as auto-pushed exception.** Standing rule "never push to master without explicit word" stays — daily-logs are the documented exception (append-only, low-risk). Memory + code still gated.
- **No autonomous agent chat with full MCP.** Cloud workflows have file access only — no Slack send, no GH PR creation, no Granola. Agents propose specific changes; Dina applies. Right tradeoff for safety.
